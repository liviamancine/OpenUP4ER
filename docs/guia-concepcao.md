# Fase de Concepção (OpenUP4ER)

A fase de **Concepção** tem como objetivo alinhar stakeholders, estabelecer a visão do sistema e identificar riscos e requisitos iniciais para guiar as próximas iterações.

---

## Method Content (átomos)

### Roles
- Stakeholder
- Engenheiro de Requisitos
- Arquiteto de Software
- Gerente de Projeto
- Desenvolvedor (participação menor, quando necessário)

### Tasks
- Definir a Visão
- Identificar Stakeholders e suas necessidades
- Identificar e priorizar requisitos iniciais
- Definir Escopo do Projeto
- Identificar riscos e estratégias de mitigação
- Planejar Iteração inicial
- Definir arquitetura candidata (nível alto)

### Work Products
- Documento de Visão
- Lista de Requisitos (alto nível)
- Lista de Stakeholders
- Lista de Riscos
- Plano de Iteração (inicial)
- Arquitetura candidata (modelo conceitual / rascunho)

### Guidance
- Templates: Documento de Visão, Lista de Riscos, Plano de Iteração
- Guias de boas práticas para elicitação inicial
- Checklists de riscos e escopo
- Exemplos de modelos de visão e arquitetura

---

## Processes (organização e fluxo)

### Discipline
- Requisitos (foco principal)
- Gerenciamento de Projeto
- Arquitetura
- Teste (mínimo, só planejamento inicial)

### Activity
- Estabelecer Visão
- Identificar Riscos
- Planejar Iteração inicial
- Esboçar Arquitetura candidata

### Capability Pattern
- Obter consenso com Stakeholders
- Elaborar Documento de Visão
- Criar Lista de Riscos
- Criar Plano de Iteração

---

## Entradas
- Objetivos de negócio
- Restrições organizacionais
- Informações de domínio
- Expectativas de stakeholders

## Saídas
- Documento de Visão (rascunho consolidado)
- Lista de Stakeholders
- Lista de Riscos principais
- Escopo inicial
- Plano de Iteração (inicial)

---

## Iteratividade
- **Iteração 1:** identificar stakeholders e alinhar visão
- **Iteração 2:** definir riscos e prioridades
- **Iteração 3:** ajustar escopo e arquitetura candidata
